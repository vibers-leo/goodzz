# MyAIPrintShop — OKR (2026 Q1)

> 최종 업데이트: 2026-03-24 by [TCC]

## Objective 1: 플랫폼 품질 안정화 및 프로덕션 신뢰성 확보

Phase 4까지 모든 핵심 기능(AI 디자인, 멀티벤더, 크리에이터, 결제)이 구현 완료된 상태. 이제 실 운영 환경에서의 안정성과 테스트 커버리지를 높여 프로덕션 신뢰성을 확보해야 함.

- KR1: E2E 테스트 커버리지 확대 — 현재: 가이드만 존재, 실제 E2E 테스트 미작성 / 목표: 주문/결제/AI 생성 핵심 플로우 3개 이상 E2E 테스트 작성
- KR2: 성능 모니터링 체계 구축 — 현재: OpenTelemetry 설정 파일 존재하나 미활성 / 목표: 주요 API 응답시간 모니터링 대시보드 구축
- KR3: Firestore 쿼리 최적화 효과 검증 — 현재: 인메모리 캐시 구현 완료 / 목표: 캐시 히트율 측정 및 P95 응답시간 500ms 이하 확인

## Objective 2: 크리에이터/벤더 생태계 실 운영 준비

크리에이터 마켓플레이스와 멀티벤더 시스템이 기술적으로 완성되었으나, 실제 크리에이터와 벤더가 입점하여 운영할 수 있는 상태로 만들어야 함.

- KR1: 와우프레스 인쇄 업체 연동 실제 주문 테스트 — 현재: 스펙 매핑 구현 완료 / 목표: 실제 샘플 주문 1건 이상 성공적 처리
- KR2: 벤더 정산 자동화 1회 이상 실행 — 현재: 정산 크론 API 구현 완료 / 목표: 테스트 벤더 대상 정산 프로세스 1사이클 완료
- KR3: SDK 임베드 기능 베타 출시 — 현재: SDK 빌드 스크립트 존재 / 목표: 외부 사이트 1개 이상에 임베드 테스트 완료

## Objective 3: 배포 인프라 전환 및 공통 패키지 정리

모노레포 내 공통 자산 정리와 NCP Docker 배포 전환을 준비한다.

- KR1: @vibers/ui 공통 컴포넌트 이관 검토 및 1차 분리 — 현재: 검토 단계 / 목표: 최소 3개 공통 컴포넌트(버튼, 모달, 카드) 이관
- KR2: NCP Docker 배포 전환 준비 — 현재: Vercel 운영 중 / 목표: Dockerfile 작성 + 스테이징 환경 구동 확인
